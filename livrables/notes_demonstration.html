Notes de demonstration - CoopLedger

Prototype de traçabilite financiere pour une cooperative agricole, deploye sur Railway et connecte a Stellar Testnet.

Objectif de la demonstration : montrer comment une cooperative peut enregistrer une transaction financiere et obtenir une preuve blockchain sous forme de hash Stellar.

Acteurs simules

  • Tresoriere de la cooperative : saisit une cotisation, une depense ou une recette dans CoopLedger.
  • Membre de la cooperative : consulte les transactions, les votes et les informations de gouvernance.
  • Observateur ou partenaire : verifie qu'une transaction possede un hash blockchain consultable sur Stellar Testnet.

Flux demontre

  1. Ouverture du prototype CoopLedger depuis le lien Railway.
  2. Consultation du dashboard : solde, cotisations, depenses, membres actifs et votes en cours.
  3. Navigation vers l'historique des transactions.
  4. Creation d'une nouvelle transaction avec un libelle et un montant valide.
  5. Envoi de la transaction vers le serveur Node.js.
  6. Scellement de la transaction sur Stellar Testnet via une operation de type manageData.
  7. Affichage du hash blockchain dans l'interface.
  8. Consultation possible du hash sur l'explorateur Stellar Testnet.

Fonctionnalites visibles

  • Dashboard synthetique de la cooperative.
  • Historique de transactions de demonstration.
  • Ajout d'une transaction scellee sur Stellar Testnet.
  • Affichage du hash blockchain apres enregistrement.
  • Page de vote simulant la gouvernance collective.
  • Page membres simulant les informations de la cooperative.

Controle des erreurs

Le prototype refuse les montants invalides avant l'enregistrement. La validation est faite dans l'interface et aussi cote serveur afin d'eviter qu'une valeur incorrecte soit envoyee sur Stellar.

Limites du prototype

  • L'historique affiche dans l'application est un historique de demonstration.
  • Les nouvelles transactions apparaissent dans la session du navigateur, mais ne sont pas encore conservees dans une base de donnees applicative.
  • La preuve permanente est le hash enregistre sur Stellar Testnet.

Prochaines ameliorations

  • Ajouter une base de donnees pour conserver l'historique complet dans l'application.
  • Relire automatiquement les transactions depuis Stellar Testnet.
  • Ajouter une authentification pour differencier tresoriere, membres et observateurs.
  • Ameliorer le module de vote avec un enregistrement blockchain des resultats.
